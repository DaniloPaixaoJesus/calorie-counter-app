# Contrato: Design System (Material 3)

## Objetivo

Padronizar componentes visuais e decisões de layout da feature 003 sem alterar regras de negócio.

## Tokens mínimos obrigatórios

- **Cores**: primária, onPrimary, surface, onSurface, onSurfaceVariant, error.
- **Tipografia**: displayLarge, headlineMedium, titleLarge, bodyLarge, labelLarge.
- **Espaçamento**: escala de 4, 8, 12, 16, 24, 32.
- **Bordas e elevação**: raio de cartões/diálogos e níveis de sombra padronizados.

## Componentes reutilizáveis

1. Card de total de calorias (destaque verde).
2. Item de refeição com ícone + descrição + calorias.
3. Cartões de escolha no fluxo "Adicionar" (texto/áudio).
4. Diálogo de confirmação de remoção.
5. Estado vazio amigável por data.

## Regras de uso

- Todos os componentes devem usar os tokens do tema.
- Não usar valores visuais hardcoded duplicados em múltiplos widgets.
- Manter tema claro como padrão.
- Não introduzir gráficos/metas/relatórios/gamificação.
